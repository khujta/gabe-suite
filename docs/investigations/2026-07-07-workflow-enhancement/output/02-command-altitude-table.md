## Cross-cutting altitude findings

1. **The "lean wrapper" convention is inverted for the lifecycle core.** commands/gabe-{teach,plan,commit,scope,push,execute}.md hold 5,594 lines of spec — the command *is* the skill. That's workable (they're loaded on demand) but it puts them outside the skill-versioning convention and makes the wrapper-skills (35-line shims) pure indirection. Recommendation: accept commands-as-specs as the convention and say so in CLAUDE.md, or move spec mass into skills — but don't stay half-and-half.
2. **The suite's real backbone is 3 commands** (commit/push/plan = 118 of ~150 observed invocations). Altitude work should optimize that hot path first (manifest-ize verify commands, evidence gate, NEXT line) before touching the long tail.
3. **Always-loaded surface:** 5,075 chars of skill descriptions + 22 slash entries. One skill (myopic) is 21 % of it. Low-cost compression available; no structural change needed.
4. **Duplication vs global rules** (locked decision §2A.5): the 80 % threshold, conventional-commit types, and model-routing tables are re-declared in suite files without referencing `~/.claude/rules/common/*`. Skeptic-verified nuance: pure "see global rule" references break on hosts without the private rules (Codex `~/.agents`); the surviving form is **state the one-line rule locally + cite the global source** — de-dup the *reasoning prose*, keep a self-contained one-liner.
