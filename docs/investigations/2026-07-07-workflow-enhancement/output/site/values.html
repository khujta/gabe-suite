The values behind the changes

Every change in the plan traces back to one of seven values. If a proposed change doesn't serve one of these, it doesn't ship — that's how the plan stays small.

V1Evidence over claims

"Done" is something you show, not something you say.

Why it's on the list: the single largest measured tax — ~32 incidents of work declared done/verified/matching without anyone rendering or comparing anything, in both projects, with no improvement trend. P1 · raw/incidents.json

Change it drivesPlan step
proof: field per phase, declared at plan time by the human1.3a
Living test set — one committed, feature-named evidence folder per feature1.3b
Side-by-side reference-vs-live render as the frontend proof form (never the showcase)1.2 / 1.3
Fresh-context evaluator grades the evidence — the builder never grades itself1.3d
Capture tool (pagecast / Playwright+ffmpeg) as a per-project manifest field; the proof GIF doubles as the demo1.3e

V2Reuse before build

Lift, don't rebuild — a stand-in shell is a defect.

Why: ~22 reconstruction incidents; five spike rounds discarded in one arc; the same lesson paid for twice, once per twin. The costliest pattern per incident in the corpus. P4 · deliverable 3 §1

Change it drivesPlan step
The lift SOP (L0 resolve → L1 inventory → L2 spike → L3 lift → L4 wire+verify)1.2
Dispatch fix — the skill fires on ad-hoc UI asks, not just /gabe-mockup1.2
gastify gets the missing reuse layer: RULES.md, screen map, CLAUDE.md hard rule0.2

gustify R4 sweep — archive the 20 settled-but-active spike stories (reuse searches must not hit dead ends)0.4

V3Enforcement over instruction

The harness remembers so the human doesn't have to.

Why: the E1–E7 contract was installed into 38 files and the incidents continued — prose the model must remember is not a gate. Deterministic checks, output contracts, and hooks are. F1 · FINDINGS.md

Change it drivesPlan step
NEXT pointer as part of each lifecycle command's output contract (not a suggestion the model may forget)0.3
One single-fire Stop hook: tree dirty + no commit this session → "next: /gabe-commit"0.3
Evidence freshness check at the gabe-commit chokepoint — WARN-and-LOG first, blocking only where warnings get ignored1.3c
suite-doctor drift check (hash repo vs installs) so silent divergence gets caught mechanically0.1

V7Human judgment preserved

Encode the detectable; keep the taste.

Why: ~40 design-taste corrections are the price of caring about the product — the variation is in the reasoning, not a procedure. Encoding them would be the overfit trap. But taste that repeats or becomes detectable graduates: gustify already turned "taste" into machine checks three times (token-class coverage, the P1–P7 layout oracle, icon params). P16 · R-K verdict

Change it drivesPlan step
