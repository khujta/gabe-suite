Docsite convention v-next — this site is the specimen

The investigation surfaced 13 doc-drift incidents and a doctrine that says evidence should feed docs. This page states the resulting convention — and the two pages next to it are built with it, so you can judge the convention by inspecting the thing it produced.

01Why a new convention (the evidence)

Three measured failures shape it. First, doc drift: 13 incidents of doc conventions re-taught or indexes silently incomplete — a docsite command index shipped missing 6 of 22 commands; the per-page "On this page" nav was re-requested twice; an established numbering pattern wasn't carried to later sections. P12 · raw/incidents.json Second, docs that narrate what the source doesn't contain: the suite's own site describes a hardening pass its skill files never received — a page with no provenance stamp can't be caught lying. F3 · raw/scan-installed-state-0.md Third, the Evidence Doctrine: verification artifacts should be the raw material of documentation, which only works if pages declare what evidence backs them. deliverable 4 §6

03The agent-facing layer (C5, in detail)

Borrowed from the agent-native-verification lead (publish state to the DOM so an agent verifies without scraping) and applied to documentation:

  • data-verify-* attributes on load-bearing elements: <main data-verify-page data-verify-claims="N">, and data-verify-claim="C2" data-verify-receipt="…" on the paragraphs that carry claims. An agent can enumerate every claim on a page with one query selector — no prose parsing.
  • A page manifest — <script type="application/json" id="gabe-page-manifest"> at the end of <main>: page, mode, generation date, source-of-truth paths, evidence paths, and the claims index with receipts. A future gabe-commit docs-audit can diff the manifest against the named sources mechanically — that's the drift-closing loop from the doc scan, made checkable.
  • What an audit would check: (1) every data-verify-claim id appears in the manifest; (2) every manifest source/evidence path exists; (3) the page's generated date is not older than its newest source of truth. Three jq-grade checks — deliberately boring. (Query the DOM, not the raw text — a regex audit false-positives on code examples like the one above; this page's first self-audit proved it.)
