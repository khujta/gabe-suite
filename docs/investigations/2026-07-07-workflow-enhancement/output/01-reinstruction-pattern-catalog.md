## Reading the table

Three structural facts explain most of the catalog:

1. **Prose contracts don't enforce.** The two biggest clusters (P1 verify-before-claim ~32, P4 reuse-not-recreate ~22) are both *already written down* — E1–E7 in every installed file, reuse rules in gabe-mockup + apps/web/CLAUDE.md + RULES.md — and kept happening. The knowledge home isn't missing; the *enforcement* home is (hook / mechanical gate / output contract).
2. **Skills don't fire on ad-hoc asks.** gabe-mockup was loaded 7 times against ~50+ UI work episodes. The reconstruction incidents almost all begin with a casual "make a spike for X" that never routes through the skill that contains the rules. Trigger/dispatch is a first-class problem (see Deliverables 3, 6).
3. **The manifest layer is half-built and asymmetric.** gustify accreted RULES.md/DESIGN.md/WEB-SCREEN-MAP/web-CLAUDE.md organically; gastify has none of these (its incidents P5/P6/P11 are exactly the gap). The per-project manifest isn't a new invention — it's naming and completing what one twin already does.
